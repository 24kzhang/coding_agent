# 健身动作网站开发可执行计划
## 一、计划目标
1. 开发仅面向电脑端的健身动作网站，直接读取`exercises-dataset/data/exercises_zh.json`全部1324条动作数据，不复制数据和GIF
2. 首页分页展示动作（每页24条），支持动作分类、目标肌肉、器材三个维度单选筛选及任意组合筛选
3. 点击动作弹出详情弹窗，展示GIF及完整数据集详情，每个动作附带独立多轮健身助手对话
4. 对话功能调用OpenAI兼容接口（https://api.longcat.chat/openai，模型LongCat-2.0），SSE流式回复，API key仅从`LONGCAT_API_KEY`环境变量读取，不写入源码/前端/文档
5. 首次对话发送完整动作数据，后续保留上下文不重复发送
6. 使用uv管理项目，提供Windows/macOS双平台启动脚本、中文README，覆盖核心功能的自动测试，实际运行测试通过后完成

## 二、前置准备
1. 确认当前目录结构：检查`exercises-dataset/data/exercises_zh.json`存在，确认数据格式正确
2. 初始化uv项目：`uv init --package`，添加依赖：fastapi, uvicorn, jinja2, python-multipart, httpx, pytest, pytest-cov

## 三、源码开发
### 3.1 后端服务（main.py）
- 静态文件服务：挂载前端资源目录，仅适配电脑端布局
- 数据接口：
  - `GET /api/exercises`：支持page, category, muscle, equipment参数，返回分页+筛选后的动作列表，每页24条
  - `GET /api/exercises/{id}`：返回单个动作完整数据
  - `GET /api/options`：返回所有可选的分类、目标肌肉、器材选项
- 对话接口：
  - `POST /api/chat/{exercise_id}`：SSE流式接口，首次请求将完整动作数据作为系统上下文发送给模型，后续请求仅保留对话上下文，从`LONGCAT_API_KEY`读取API key，调用LongCat-2.0模型

### 3.2 前端页面（templates/index.html, static/app.js, static/style.css）
- 仅适配电脑端，不使用响应式移动端布局
- 首页：分页控件 + 三个筛选下拉框 + 动作卡片网格（每行4-6个卡片，适合电脑屏幕）
- 详情弹窗：展示动作GIF、全部字段，底部附带健身助手对话窗口，支持多轮输入，流式展示回复

### 3.3 启动脚本
- `start.sh`：macOS启动脚本，内容：`export LONGCAT_API_KEY=your_key && uv run uvicorn main:app --host 0.0.0.0 --port 8000`
- `start.bat`：Windows启动脚本，内容：`set LONGCAT_API_KEY=your_key && uv run uvicorn main:app --host 0.0.0.0 --port 8000`

## 五、自动测试开发（tests/test_main.py）
覆盖以下核心场景：
1. 数据接口测试：分页正确性、单筛选/组合筛选正确性、返回总数正确
2. 详情接口测试：返回完整动作数据，字段匹配
3. 对话接口测试：SSE流式响应正确，首次请求包含动作数据，后续上下文保留
4. 媒体测试：GIF链接可访问
5. 异常测试：参数错误、动作不存在时的错误处理
